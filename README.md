# 🚗 基于 LLM 的汽车系统需求文档自动化提取工具

> 利用大语言模型 (LLM) 与 Python 实现的非结构化文档解析与技术指标结构化流水线。

## 📖 项目简介
在传统的系统开发与测试流程中，从长篇幅的 PDF 需求文档中提取并核对技术指标是一项耗时且极易出错的重复性任务。
本项目旨在通过自动化脚本与大模型技术，实现从非结构化的汽车零部件测试文档到结构化 Excel 数据表的**端到端自动化解析**，大幅提升系统需求分析环节的效率。

## 📂 项目架构与模块说明 (核心亮点)
本项目采用了良好的模块化设计，解耦了业务逻辑与底层代码：

* **`main.py`**：系统主调度入口，负责串联文档读取、大模型提取、清洗对齐与数据导出的全流程。
* **`pdf_parser.py`**：PDF 解析模块。负责读取原始设计文档，并进行初步的文本块清洗与格式化。
* **`llm_extractor.py`**：核心 AI 提取模块。深度集成大模型 API，通过定制化的 Prompt（提示词工程），精准抽取光学性能、电气环境等各项技术参数。
* **`term_alignment.py`**：术语对齐与数据清洗模块。用于处理 LLM 输出的非标准格式，确保指标类型（如 Performance, Environment）的准确映射。
* **`term_dict.json`**：**领域知识字典**。将汽车行业的专业术语配置化，使得工具不依赖于特定的硬件代码，只需修改字典即可跨项目复用。
* **`exporter.py`**：数据持久化模块。将对齐后的结构化数据一键导出为标准化的 Excel 表格。

## ✨ 核心技术栈
- **语言/脚本**: Python 3.x
- **数据处理**: Pandas
- **AI 驱动**: 兼容大语言模型 API (LLM API) / Prompt Engineering
- **文档处理**: PyMuPDF / pdfplumber

## 💡 研发思考
本项目的核心价值在于**利用 AI 打造部门级效率工具**。通过字典映射的设计（`term_dict.json`），系统成功将复杂的业务领域知识与纯粹的软件逻辑隔离开来，具有极高的工程落地价值与可扩展性。
